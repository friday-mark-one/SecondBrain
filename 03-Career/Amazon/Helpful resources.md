### Slack thread for git push remote

You can push your code to a share branch `git push share HEAD:feature_name`

So there are multiple repositories per package. There's the main one that we push our code to labeled `origin`, then there's the `share` where you can push to but everyone else can only read, then there's `backup` which is, AFAIK, only accessible to you.

`git remote -v` to view them.

The `Branches` dropdown on [code.amazon.com](http://code.amazon.com/) is searchable so you can use that to find an exact branch, but if you scroll down they're ordered - Official branches (`origin`), Your shared branches (`share`), And then everyone else's shared branches sorted by username

If you hover over someones shared branch in [code.amazon.com](http://code.amazon.com/) it'll have a popup with a command you can copy/paste to pull down the code, such as - `git add-share-remote baaskab ; git fetch baaskab && git checkout baaskab/feature_name`
